# Gemini Guidelines

All foundational mandates and AI agent guidelines for this project are centralized in `CLAUDE.md`.

Gemini MUST adhere to the branching, commit message, and PR standards defined in `CLAUDE.md`.

## Key Mandates (from CLAUDE.md)

- **Atomic Commits:** One logical change per commit.
- **Commit Messages:** Capitalized, imperative mood, no type prefixes (e.g. "Add feature", NOT "feat: Add feature").
- **PR Titles:** Use `[feature/<name>]` or `[chore]` prefixes.
- **PR Descriptions:** Always use the pull request template.
- **Testing:** Always run tests, linting, and type checking before pushing.
